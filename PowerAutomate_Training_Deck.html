⚠️ Critical prerequisite. Power Automate's "List rows present in a table" action can ONLY read data that is formatted as an official Excel Table — not a plain spreadsheet range. If this step is skipped, the flow will run but will not read any donor rows.

Step-by-step: format your data as a Table

1
Open your own copy of the "Clean No Frills Donor Excel Sheet"
This must be your personal file. It must be saved to your OneDrive for Business — not your computer's Desktop or Downloads folder. Power Automate can only access files stored in OneDrive or SharePoint.
☐
2
Confirm Row 1 contains these exact column headers
Your headers must read: Full Name · Email · Top Designation Name
Spelling, spaces, and capitalisation must match exactly. Power Automate uses these names to identify the correct columns.
☐
3
Click any cell inside your donor data (not outside it)
Click anywhere inside the rows of data — for example cell A2 or B5. Do not click on an empty cell or a cell outside the data range.
☐
4
Press Ctrl + T on your keyboard
A "Create Table" dialog box will appear. Check that the range shown is correct and that "My table has headers" is ticked. Click OK. Your data will now display blue striped rows — this confirms it is an Excel Table.
☐
5
Name your Table "DonorTable"
With the table selected, click the Table Design tab in the Excel ribbon. In the top-left box labelled "Table Name," delete the default name and type DonorTable. Press Enter. Save the file (Ctrl + S).

Common errors and fixes

❌ "No tables found" in Table dropdown
Your Excel data is not formatted as a Table. Go to Excel → click inside the data → press Ctrl+T → click OK → save the file → return to Power Automate.
❌ "InvalidTemplate" or "Expression error"

❌ Drafts created but "To" field is blank
The Email column dynamic content was not selected correctly. Edit the Create Draft action → click the To field → re-select the Email dynamic content value.

Important known issues and fixes

⚠️ OSU email addresses require angle brackets
When setting the To field, OSU email addresses must be wrapped in angle brackets: <swart.50@osu.edu>. Without the brackets, the action will show a validation error on OSU accounts specifically.
⚠️ Use body/content — not body
When referencing the original email's body in the Create Draft action, always use the expression path body/content. Using body alone returns a complex object that causes an HTML encoding error in the draft.
⚠️ toRecipients is an array, not a string
The recipient field is an array of objects. You must use first() to get the first recipient and navigate the nested object to reach the email address. See the expression on the previous slide.

Flow 5 of 6 — Expressions
Meeting Follow-Up — Key Expressions
Condition — keyword check (meeting title)
Expression
or(contains(toLower(triggerOutputs()?['body/subject']), 'donor'), contains(toLower(triggerOutputs()?['body/subject']), 'prospect'), contains(toLower(triggerOutputs()?['body/subject']), 'visit'), contains(toLower(triggerOutputs()?['body/subject']), 'meeting')) expression
Why toLower()
Makes the check case-insensitive. Without it, "Donor" would not match "donor." This was a known confusion point during build.
Condition — confirm meeting has ended
Expression
greaterOrEquals(utcNow(), triggerOutputs()?['body/end']) expression
Known issue
The end time field path is body/end. Using body/endTime or end/dateTime will cause this condition to always return false — the persistent issue from our troubleshooting sessions.
Delay until — 8 AM Eastern next morning
Expression
addHours(startOfDay(addDays(utcNow(), 1)), 12) expression
Explanation
Calculates midnight tomorrow (UTC) then adds 12 hours to reach 8 AM Eastern (UTC-4 in EDT). Paste in the "Timestamp" field of the Delay until action.
Condition — filter out OSU attendees
Expression
not(endsWith(items('Apply_to_each')?['emailAddress/address'], '@osu.edu')) expression
Where used
Condition wrapping the Create Draft action inside an Apply to each loop over the meeting attendees. Only creates a draft if the expression returns true (non-OSU address).

Flow 6 of 6 — Expressions
Meeting Prep Digest — Key Expressions
Filter — events starting today (lower bound)
Expression
greaterOrEquals(item()?['start/dateTime'], startOfDay(utcNow())) expression
Where used
First condition in a Filter array action on the Get events results
Filter — events starting today (upper bound)
Expression
less(item()?['start/dateTime'], addDays(startOfDay(utcNow()), 1)) expression
Why this approach
endOfDay() does not exist in Power Automate — using it causes a build error. This expression replaces it by checking that the event starts before the beginning of tomorrow.
Compose — convert UTC time to Eastern
Expression
convertTimeZone(item()?['start/dateTime'], 'UTC', 'Eastern Standard Time', 'h:mm tt') expression
Critical note
The timezone string must be exactly 'Eastern Standard Time' — any variation will throw an error. This produces output like "10:00 AM".
Compose — build digest line per meeting
Expression
concat(outputs('Convert_Time'), ' ET — ', item()?['subject']) expression
Result
Produces: 10:00 AM ET — Donor Meeting: Williams Foundation
